# 🎙️ 議事録レコーダー(録音 → NotebookLM → 議事録 全自動)

ブラウザで会議を録音すると、音声が**自分のNotebookLM**に自動アップロードされ、議事録が生成されて画面に表示されるツールです。

- 録音 → アップロード → 議事録生成 → 表示まで**ボタン1つ**
- 議事録はNotebookLMのノートにも自動保存
- 録音ファイルは手元のMacにも自動ダウンロード(保険)
- データは自分のMacと自分のGoogleアカウントの間だけで完結(外部サーバーなし)

## 必要なもの

- Mac(Apple Silicon / Intel どちらも可)
- Python 3.10以上(なければ https://www.python.org/downloads/ から)
- NotebookLMが使えるGoogleアカウント(慶應の @keio.jp でOK)

## インストール(初回のみ・5分)

ターミナル(アプリ一覧の「その他」→「ターミナル」)を開いて、以下を1行ずつ実行:

```bash
git clone https://github.com/ikuma0245/Noteboolm_gijitotte.git
cd Noteboolm_gijitotte
./setup.sh
```

> gitがない場合は、GitHubページの「Code」→「Download ZIP」でダウンロードして展開し、そのフォルダで `./setup.sh` でもOK。

## 使い方(毎回)

```bash
cd Noteboolm_gijitotte
./start.sh
```

→ ブラウザが自動で開きます(http://localhost:8765)

1. **初回のみ**: 画面の「🔑 再認証」ボタンを押す → 開いたブラウザでGoogleログイン(keio.jpアカウント)
2. 会議名を入力し、アップロード先ノートブックを選択(新規作成も可)
3. 「録音開始」→ 会議 → 「録音停止 & 送信」
4. 数分待つと議事録が画面に表示される(コピー / .md保存 / NotebookLMで開く、が可能)

終了はターミナルで `Ctrl+C`。

## ☁️ 共有保存と会議一覧(Firebase・任意)

`firebase-key.json` をこのフォルダに置くと、議事録生成のたびに**録音音声と議事録が共有のFirebaseにも保存**され、**http://localhost:8765/admin** の「会議一覧」で寮の全会議を後から検索・再生・閲覧できます。鍵が無い場合、この機能は自動でオフになりアプリは通常通り動きます。

### 管理者(初回のみ)のFirebase設定

1. https://console.firebase.google.com でプロジェクトを作成
2. 「Firestore Database」→ データベースを作成(ロケーションは `asia-northeast1`、本番モードでOK — アクセスはサービスアカウント経由のみなのでルールは触らなくてよい)
3. 「Storage」→ 有効化(※新規プロジェクトはBlaze(従量課金)プランへのアップグレードが必要。無料枠内なら実質0円)
4. ⚙️ プロジェクト設定 → サービスアカウント → 「新しい秘密鍵の生成」→ ダウンロードしたJSONを `firebase-key.json` という名前でこのフォルダ直下に置く

### 寮生への配布

- `firebase-key.json` を **AirDropや手渡しなど非公開の方法で**共有し、各自のフォルダ直下に置いてもらう
- **この鍵をGitHubやLINEグループ等に公開しないこと**(Firebaseへのフルアクセス権そのものです)。`.gitignore` 済みなので誤コミットはされません
- 鍵を持っている人は全員、会議一覧の閲覧・削除ができます(寮内の信頼ベースの運用想定)

## よくあるトラブル

- **「認証が切れています」と出る**: 画面の「🔑 再認証」ボタンを押すだけでOK。まず裏でCookieを更新し、ダメならログイン用ブラウザが自動で開きます
- **マイクが使えない**: ブラウザのマイク許可ダイアログで「許可」を選ぶ。システム設定 → プライバシーとセキュリティ → マイク も確認
- **アップロードが遅い**: 長時間録音は処理に数分かかります(15分でタイムアウト)
- **議事録の形式を変えたい**: 画面の「議事録プロンプトをカスタマイズ」を編集(自動保存されます)
- **`./setup.sh` が動かない**: `chmod +x setup.sh start.sh` を実行してから再試行

## 仕組みと注意事項

- 非公式ライブラリ [notebooklm-py](https://github.com/teng-lin/notebooklm-py) でNotebookLMの内部APIを操作しています
- 認証はあなたのブラウザログインのCookieを**あなたのMacの中だけ**で使います(どこにも送信されません)
- **注意**: 非公式APIのため、Googleの仕様変更で突然動かなくなる可能性・利用規約上のリスクがあります。自己責任・個人利用の範囲で使ってください
- NotebookLM無料枠の上限: 議事録生成は1日50回まで、1ファイル200MBまで(普通の使い方なら気になりません)
